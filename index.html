المحور الأول

ما هو الذكاء الاصطناعي؟

الذكاء الاصطناعي هو مجال في علوم الحاسب يهدف إلى بناء أنظمة تستطيع تنفيذ مهام كانت تحتاج في السابق إلى ذكاء بشري، مثل التعلّم، والاستدلال، وفهم اللغة، وتحليل البيانات، وتقديم اقتراحات مفيدة. بصياغة أبسط: هو جعل الحاسوب لا ينفذ الأوامر فقط، بل يتعامل مع المعلومات بقدر من “الفهم العملي” لاتخاذ مخرجات أفضل.

مثال بسيط

فلتر البريد المزعج يتعلم من الرسائل السابقة ليقرر: هل هذه الرسالة مفيدة أم مزعجة؟

مثال عملي

مساعد برمجي يقترح إصلاح خطأ في الكود بناءً على فهمه للمشروع والملفات.

فكرة مهمة

الذكاء الاصطناعي مظلة كبيرة، وتحتها يأتي التعلم الآلي، والتعلم العميق، والنماذج اللغوية الكبيرة.

المحور الثاني

ما هو LLM؟

الـ LLM أو Large Language Model هو نموذج متقدم من نماذج الذكاء الاصطناعي مُدرَّب على كميات هائلة من البيانات النصية، بحيث يتعلم أنماط اللغة ويستطيع فهم اللغة الطبيعية وتوليدها لأداء مهام متعددة مثل الشرح، والتلخيص، والكتابة، والبرمجة، والتحليل. وبصياغة مبسطة: هو “محرك تنبؤ لغوي ضخم” يتوقع التسلسل الأنسب للكلمات اعتمادًا على ما تعلمه من البيانات.

تخيل أن لديك متدربًا قرأ مليارات الجمل، ثم صار بارعًا في توقع الكلمة التالية وصياغة إجابات مفهومة. هذا هو قلب الـ LLM: بارع في اللغة، لكنه ليس بالضرورة نظام تنفيذ مستقل بذاته.
أمثلة على النماذج اللغوية الكبيرة: GPT من OpenAI، وClaude من Anthropic، وGemini من Google. الفكرة هنا ليست اسم النموذج فقط، بل كونه محرك الفهم والتوليد اللغوي داخل التطبيق أو الوكيل.
الـ LLM وحده لا يعني تلقائيًا أنه سيقرأ ملفاتك، أو يشغّل أوامر، أو يتصل بخدمة خارجية. لكي يفعل هذا عمليًا يحتاج إلى أدوات، وصلاحيات، وسياق، وربما بروتوكول مثل MCP يفتح له الوصول المنظم إلى الأنظمة الخارجية.
المحور الثالث

ما هو وكيل الذكاء الاصطناعي؟ وما الفرق بينه وبين النموذج؟

بحسب دليل OpenAI، الوكلاء هم تطبيقات تخطط، وتستدعي الأدوات، وتتعاون بين تخصصات مختلفة، وتحتفظ بقدر كافٍ من الحالة لإتمام أعمال متعددة الخطوات. كما تصف Google Cloud وكلاء الذكاء الاصطناعي بأنهم أنظمة برمجية تستخدم الذكاء الاصطناعي لتحقيق أهداف وإنجاز مهام نيابة عن المستخدم، مع قدر من الاستقلالية والتخطيط والذاكرة.

النموذج Model

  • يفهم ويولد النص
  • يجيد الشرح والتحليل والاقتراح
  • لا ينفذ العمل الخارجي وحده غالبًا
  • يشبه “العقل اللغوي”

الوكيل Agent

  • يستخدم النموذج كمحرك تفكير
  • يخطط على عدة خطوات
  • يستدعي أدوات ويقرأ ملفات ويتخذ إجراءات
  • يشبه “العقل + اليد + الذاكرة + الأدوات”
مثال يوضح الفرق بسرعة

إذا قلت لنموذج لغوي: “اكتب لي خطة لإصلاح مشروع برمجي” فسوف يعطيك خطة ممتازة. لكن إذا قلت لوكيل برمجي: “افحص المشروع وحدد الخطأ ثم اقترح التعديل ونفذه بعد موافقتي” فهنا الوكيل يمكنه قراءة الملفات، واستخدام الأدوات، وربما تشغيل أوامر واستخدام MCP، ثم العودة بنتيجة عملية.

التطبيق العملي: OpenCode وMCP وSkills

هذه الوحدة تركّز على الاستخدام الفعلي: ما هو OpenCode، كيف تثبّته، كيف تربطه بخوادم MCP، وكيف تضيف له مهارات قابلة لإعادة الاستخدام.

المحور الرابع

مدخل إلى OpenCode

OpenCode يعرّف نفسه رسميًا بأنه وكيل برمجي مفتوح المصدر، ومتوفر على شكل واجهة طرفية Terminal، وتطبيق Desktop، وإضافة للـ IDE. هذه نقطة مهمة جدًا: OpenCode ليس مجرد نموذج لغوي، بل بيئة عمل لوكيل برمجي يمكنه فهم مشروعك، واستخدام الأدوات، والانتقال بين أوضاع مثل build وplan.

Terminal

تجربة قوية داخل الطرفية للمطورين الذين يحبون السرعة والتحكم.

Desktop

واجهة رسومية رسمية بنسخة Beta لأنظمة متعددة.

IDE Extension

إمكانية العمل بجانب VS Code وCursor وZed وغيرها.

ماذا يفعل OpenCode داخل المشروع؟

بعد الانتقال إلى مجلد المشروع وتشغيل OpenCode، يمكنك تنفيذ /init ليحلل المشروع وينشئ ملف AGENTS.md في الجذر ليساعده على فهم البنية وأنماط العمل داخل الكود. كجزء من الإعداد العملي داخل المشروع.

المحور الخامس

تثبيت OpenCode على الديسكتوب

صفحة التحميل الرسمية توضّح أن OpenCode Desktop (Beta) متاح لأنظمة macOS وWindows وLinux. كما يذكر المستودع الرسمي أن التطبيق يمكن تنزيله مباشرة من صفحة التحميل أو من صفحة الإصدارات على GitHub.

1
افتح صفحة التحميل الرسمية

الرابط الرسمي: opencode.ai/download

2
اختر نسخة النظام

المتوفر رسميًا: macOS (Apple Silicon / Intel)، Windows x64، Linux بصيغتي .deb و .rpm.

3
بديل سريع على macOS

يوضح المستودع الرسمي إمكانية التثبيت عبر Homebrew باستخدام cask الخاص بالنسخة المكتبية.

brew install --cask opencode-desktop

حتى لو كنت ستستخدم Desktop، من المفيد أن تعرف طرق تثبيت OpenCode الأساسية أيضًا، لأن كثيرًا من الاستخدامات العملية تبدأ من سطر الأوامر.

# أسهل طريقة
curl -fsSL https://opencode.ai/install | bash

# npm
npm install -g opencode-ai

# macOS / Linux عبر Homebrew
brew install anomalyco/tap/opencode

# Windows
choco install opencode
scoop install opencode

هذه الأوامر مذكورة في المقدمة الرسمية والمستودع الرسمي.

cd /path/to/project
opencode
/init

بعد الإعداد، الخطوة العملية المعتادة هي الدخول إلى مجلد المشروع، ثم تشغيل OpenCode، ثم تنفيذ /init ليحلل المشروع وينشئ ملف AGENTS.md.

المحور السادس

ما هو MCP؟

MCP اختصار لـ Model Context Protocol، وهو معيار مفتوح المصدر لربط تطبيقات الذكاء الاصطناعي بالأنظمة الخارجية مثل الملفات المحلية، وقواعد البيانات، ومحركات البحث، والواجهات البرمجية، وسير العمل الجاهز. ويمكن فهمه ببساطة على أنه منفذ موحّد يربط تطبيقات الذكاء الاصطناعي بالأدوات والأنظمة الخارجية بدل بناء تكامل منفصل لكل أداة.

لماذا هو مهم؟

لأنه يقلل التعقيد ويجعل بناء التكامل مع الأدوات الخارجية أكثر توحيدًا للمطور والوكيل والمستخدم النهائي.

كيف يفيد الوكيل؟

يعطيه وصولًا منظمًا إلى أدوات وبيانات حقيقية بدل الاعتماد على النص فقط.

كيف تفهمه بسرعة؟

النموذج يفكر، والوكيل يقرر، وMCP يفتح له الباب للوصول إلى الأنظمة الخارجية.

ما هو MCP Server؟

خادم MCP هو برنامج يقدّم قدرات محددة لتطبيقات الذكاء الاصطناعي عبر واجهة معيارية، ويمكن أن يوفّر Tools وResources وPrompts.

  • Tools: وظائف يناديها النموذج مباشرة لتنفيذ أفعال مثل استدعاء API أو تعديل ملفات.
  • Resources: مصادر بيانات للقراءة فقط مثل ملفات أو وثائق أو مخططات قاعدة البيانات.
  • Prompts: قوالب جاهزة تنظّم طريقة استخدام الأدوات والموارد في مهام متكررة.
المحور السابع

طريقة ربط الوكيل في MCP داخل OpenCode

يمكنك إضافة أدوات خارجية إلى OpenCode باستخدام MCP، سواء كانت خوادم محلية أو بعيدة، وبعد إضافتها تصبح هذه الأدوات متاحة للوكيل مع بقية الأدوات المدمجة.

1
افتح ملف إعداد OpenCode

يتم تعريف خوادم MCP داخل إعداد OpenCode تحت المفتاح mcp.

2
أضف خادم MCP باسم فريد

كل خادم يحتاج اسمًا مميزًا، مع نوعه local أو remote، ثم الأمر أو الرابط المناسب.

3
اطلب من الوكيل استخدامه بالاسم

يمكن الإشارة إلى خادم MCP باسمه مباشرة داخل البرومبت، مثل: “use the mcp_everything tool”.

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "mcp_everything": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-everything"]
    }
  }
}

هذا المثال يوضح الشكل العملي المتوقع لإعداد خادم MCP داخل OpenCode.

مثال برومبت عملي بعد الربط

مثال تعليمي مبسط:
“افحص المهمة الحالية، وإذا احتجت أدوات إضافية فاستخدم mcp_everything tool ثم لخّص لي النتائج بالعربي.”
الفكرة هنا ليست النص نفسه حرفيًا، بل أن الوكيل يستطيع الإشارة إلى أداة MCP التي عرّفتها بالاسم داخل الإعداد.

مثال عملي: إضافة Playwright MCP عبر mcpservers.org

إذا كنت تريد أن تجعل الوكيل يتعامل مع المتصفح بشكل فعلي، فواحد من أفضل الأمثلة هو Playwright MCP. الفكرة ببساطة: تبحث عنه داخل mcpservers.org، ثم تضيفه إلى إعداد OpenCode، وبعدها يستطيع الوكيل فتح الصفحات، والتنقل، والنقر، وملء النماذج، واستخراج البيانات من المواقع بشكل منظم.

هذا المثال مناسب في حالات مثل: اختبار صفحة ويب، تسجيل الدخول إلى بيئة تجريبية، جمع معلومات من لوحة تحكم، أو تنفيذ خطوات متكررة داخل المتصفح بدل تنفيذها يدويًا. 

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "playwright": {
      "type": "local",
      "command": ["npx", "-y", "@playwright/mcp@latest"]
    }
  }
}

بعد الإضافة يمكنك توجيه الوكيل بأوامر مباشرة مثل: 

استخدم أداة playwright وافتح الموقع المطلوب، ثم انتقل إلى الصفحة الرئيسية،
واستخرج العناوين الرئيسية، وإذا وجدت زر تسجيل دخول فاذكر مساره وخطوات الوصول إليه.

أو:

استخدم playwright لفتح لوحة التحكم التجريبية، ثم التقط تسلسل الخطوات اللازمة
لإنشاء مستخدم جديد، واكتبها لي خطوة بخطوة.

في التطبيق العملي، يفضّل أن تكون تعليماتك محددة: ما الصفحة التي تريد فتحها، وما البيانات التي تريد استخراجها، وهل المطلوب فقط الفحص أم التفاعل الكامل مع العناصر داخل الصفحة.

المحور الثامن

ما هو mcpservers.org؟

الموقع يعرّف نفسه بأنه مجموعة/دليل لخوادم Model Context Protocol. عمليًا: هو مكان لاكتشاف خوادم MCP الرسمية والمجتمعية حسب الفئة والاستخدام، مثل التصفح الآلي، وسحب الويب، وGitHub، وقواعد البيانات، وRAG، وPDF، وغير ذلك.

ما فائدته لك؟

بدل أن تبحث عشوائيًا عن خادم MCP مناسب، تبدأ من دليل منظم يجمع الخيارات المتاحة.

متى تستخدمه؟

عندما تريد إضافة قدرة جديدة لوكيلك، مثل GitHub أو متصفح أو مستندات أو بحث ويب.

كيف تفكر فيه؟

يشبه “متجر اكتشاف” أو “فهرس أدوات” لوكلاء الذكاء الاصطناعي المتوافقين مع MCP.

أمثلة على فئات موجودة في الموقع

من الفئات الظاهرة على الصفحة الرئيسية: Browser Automation، Web Scraping، RAG، OpenAPI، PDF، Coding Agent MCP. وهذا يساعدك على اختيار الخادم المناسب حسب السيناريو العملي الذي تريد دعمه في وكيلك.

المحور التاسع

كيف تثبّت المهارات Skills للوكيل في OpenCode؟

تسمح Agent Skills للوكيل باكتشاف تعليمات قابلة لإعادة الاستخدام من المشروع أو من مجلد المستخدم، ويتم تحميلها عند الحاجة بواسطة أداة skill. وهذا يعني أنك تستطيع تجهيز “مهارات جاهزة” مثل: مراجعة Pull Request، أو تجهيز release notes، أو أسلوب موحد للتوثيق.

يبحث OpenCode عن المهارات في عدة مسارات، وأهمها:

  • .opencode/skills/<name>/SKILL.md داخل المشروع
  • ~/.config/opencode/skills/<name>/SKILL.md على مستوى المستخدم
  • كما يدعم مسارات متوافقة مع .claude/skills و .agents/skills.

كل مهارة يجب أن تبدأ بـ YAML frontmatter يتضمن على الأقل name وdescription. كما يجب أن يطابق الاسم اسم المجلد وأن يكون بصيغة lowercase مع شرطات.

---
name: git-release
description: Create consistent releases and changelogs
license: MIT
compatibility: opencode
metadata:
  audience: maintainers
  workflow: github
---

## What I do
- Draft release notes from merged PRs
- Propose a version bump
- Provide a copy-pasteable gh release create command

## When to use me
Use this when you are preparing a tagged release.
Ask clarifying questions if the target versioning scheme is unclear.

هذا المثال يوضح شكل المهارة الأساسية وطريقة كتابتها.

يمكنك التحكم في المهارات المسموح بها أو المرفوضة أو التي تحتاج موافقة باستخدام إعدادات permission.skill داخل opencode.json. وتدعم الأنماط wildcard مثل internal-*. 

{
  "permission": {
    "skill": {
      "*": "allow",
      "pr-review": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
كيف يحمّل الوكيل المهارة؟

يعرض OpenCode المهارات المتاحة داخل وصف أداة skill، ثم يستطيع الوكيل تحميل المهارة باستدعاء مثل skill({ name: "git-release" }).

مثال عملي: مهارة للتعامل مع Excel Sheets

لو أردت أن تجعل الوكيل ينشئ ملفات Excel بشكل متكرر بأسلوب موحد، فالحل الأفضل هو تجهيز مهارة مخصصة. هذه المهارة يمكن أن تعطيه قواعد ثابتة مثل: إنشاء ملف .xlsx، تنسيق صف العناوين، تفعيل الفلاتر، ضبط عرض الأعمدة، وتجميد الصف الأول.

أنشئ مجلدًا باسم excel-sheet-builder ثم ضع داخله ملف SKILL.md بهذا المحتوى: 

---
name: excel-sheet-builder
description: Create clean Excel spreadsheets with headers, formatting, filters, and summary sheets
license: MIT
compatibility: opencode
metadata:
  audience: analysts
  workflow: excel
---

## What I do
- Create .xlsx files from user requirements
- Add header formatting and freeze the first row
- Auto-size columns and enable filters
- Create more than one sheet when needed
- Add a summary sheet if the task requires totals or KPIs

## When to use me
Use this skill when the user asks for an Excel workbook, sales tracker, attendance sheet,
inventory file, budget workbook, or any structured spreadsheet.

## How I should behave
1. Ask for missing columns only if they are not clear.
2. Build the workbook in a clean business-ready format.
3. Use meaningful sheet names.
4. Save the final file with the requested filename.
5. Summarize what sheets and columns were created.

بعد ذلك تستطيع أن تأمر الوكيل بشكل مباشر، مثل: 

استخدم مهارة excel-sheet-builder وأنشئ ملف Excel باسم sales-report.xlsx
يحتوي على شيت للمبيعات اليومية بالأعمدة التالية:
التاريخ، اسم المنتج، الكمية، سعر الوحدة، إجمالي البيع.

أضف شيت ثاني باسم summary يلخص إجمالي المبيعات وعدد العمليات،
واجعل صف العناوين مميزًا مع تفعيل الفلاتر وتجميد الصف الأول.

بهذه الطريقة لا تكرر التعليمات نفسها كل مرة. بدل ذلك، تصبح طريقة بناء ملفات Excel جزءًا من سلوك الوكيل نفسه، ويمكنك إعادة استخدام المهارة في أي مشروع أو مجلد مناسب.

أخطاء شائعة في تثبيت المهارات
  • اسم الملف ليس SKILL.md بحروف كبيرة
  • عدم وجود name أو description في frontmatter
  • اسم المهارة لا يطابق اسم المجلد
  • المهارة محجوبة بسبب الصلاحيات

إذا لم تظهر المهارة داخل الوكيل، راجع هذه النقاط أولًا قبل البحث عن سبب آخر.

الاختبار النهائي

يجب على المتدرب كتابة اسمه أولًا ثم تقديم الاختبار. لكل مستخدم محاولة واحدة فقط على هذا المتصفح.

تعليمات الاختبار:
  • أدخل الاسم الكامل قبل البدء.
  • أجب عن جميع الأسئلة ثم اضغط إرسال الاختبار.
  • بعد الإرسال سيتم قفل المحاولة ولن يمكن إعادة التقديم من نفس المتصفح.

4) ما الذي يميز الوكيل عن النموذج فقط؟

5) أين يتم تعريف خوادم MCP في OpenCode؟

6) ما فائدة mcpservers.org؟

7) ما الأمر الذي يحلل المشروع ويُنشئ AGENTS.md؟
